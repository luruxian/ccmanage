# 设计系统检查清单

## 概述

这个检查清单用于确保所有新设计和代码修改都符合设计系统规范。请在代码审查和设计评审时使用。

## 设计评审检查清单

### 🎨 色彩使用
- [ ] 所有颜色都来自设计令牌 (`src/design-tokens.ts`)
- [ ] 没有使用硬编码颜色值 (如 `#3b82f6`)
- [ ] 颜色对比度符合WCAG 2.1 AA标准
- [ ] 深色模式下的颜色表现正常
- [ ] 语义颜色使用正确 (成功、警告、错误)

### 📐 间距系统
- [ ] 所有间距使用标准值 (4px的倍数)
- [ ] 没有使用任意间距值 (如 `p-[10px]`)
- [ ] 间距层次清晰，视觉平衡
- [ ] 响应式间距适配良好

### 🔤 字体系统
- [ ] 字体族使用标准设置 (Inter, JetBrains Mono)
- [ ] 字号使用标准值 (xs, sm, base, lg, xl, 2xl, 3xl)
- [ ] 字重使用标准值 (normal, medium, semibold, bold)
- [ ] 行高使用标准值 (tight, normal, relaxed)

### 🔲 圆角系统
- [ ] 圆角使用标准值 (none, sm, base, md, lg, xl, 2xl, full)
- [ ] 没有使用任意圆角值
- [ ] 圆角使用一致，符合组件语义

### 🌫️ 阴影系统
- [ ] 阴影使用标准层级 (soft, medium, large)
- [ ] 阴影使用符合元素重要性
- [ ] 深色模式下的阴影表现正常

### 🎭 动画系统
- [ ] 动画持续时间使用标准值 (fast, normal, slow)
- [ ] 缓动函数使用标准设置
- [ ] 动画性能良好，不会造成卡顿

## 组件使用检查清单

### 🔘 按钮组件
- [ ] 使用正确的按钮变体 (default, secondary, outline, ghost, link)

- [ ] 表单验证状态清晰
- [ ] 错误提示明确
- [ ] 焦点状态完整

### 🧭 导航组件

- [ ] 当前状态指示明确
- [ ] 导航交互流畅
- [ ] 移动端导航适配

## 代码质量检查清单

### 🔧 设计令牌使用
- [ ] 导入设计令牌 (`import { colors, spacing } from '@/design-tokens'`)
- [ ] 使用TypeScript类型确保类型安全
- [ ] 没有直接使用CSS变量 (如 `var(--primary)`)

### 📱 响应式设计
- [ ] 移动端布局合理 (sm: 640px)
- [ ] 平板端布局合理 (md: 768px)
- [ ] 桌面端布局合理 (lg: 1024px, xl: 1280px)
- [ ] 断点使用正确

### ♿ 可访问性
- [ ] 颜色对比度足够 (4.5:1 for normal text)
- [ ] 键盘导航支持完整
- [ ] 焦点状态清晰可见
- [ ] 屏幕阅读器支持良好
- [ ] 图片有alt文本
- [ ] 表单有适当标签

### 🎯 性能
- [ ] 没有不必要的重渲染
- [ ] 图片优化良好
- [ ] 动画性能流畅
- [ ] 代码分割合理

## 设计系统违规示例

### ❌ 违规代码示例
```typescript
// 硬编码颜色
<div className="text-[#3b82f6]">文本</div>

// 任意间距
<div className="p-[10px]">内容</div>

// 直接使用CSS变量
<div style={{ color: 'var(--primary)' }}>文本</div>
```

### ✅ 合规代码示例
```typescript
// 使用Tailwind类
<div className="text-primary">文本</div>

// 使用标准间距
<div className="p-4">内容</div>

// 使用设计令牌
import { colors } from '@/design-tokens'
<div style={{ color: colors.primary[500] }}>文本</div>
```

## 自动化检查

### ESLint规则
- 检测硬编码颜色值
- 检测任意间距值
- 检测直接CSS变量使用

### Stylelint配置
- 强制使用设计令牌
- 检测颜色对比度
- 验证CSS变量使用

### Git Hooks
- 提交前自动运行设计系统检查
- 阻止不符合规范的代码提交

## 审查流程

### 1. 自检
开发者在提交代码前完成自检

### 2. 代码审查
审查者使用此检查清单进行审查

### 3. 设计评审
设计师验证视觉实现符合设计规范

### 4. 自动化检查
CI/CD流水线运行自动化检查

## 问题处理

### 发现违规
1. 在代码审查中标记问题
2. 提供合规代码示例
3. 要求开发者修复

### 争议处理
1. 讨论设计系统是否满足需求
2. 考虑扩展设计系统
3. 记录决策和理由
